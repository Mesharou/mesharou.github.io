IV - Set up and run a regional configuration

We are going to use the Python version of the Croco tools to generate a grid, initial and boundary files, as well as atmospheric data to run a regional simulation in the region of Benguela.

The detailed explanations are available here. The main steps and the instructions specific to the IUEM computers are explained below.

Download the croco tools

see tuto.env.html

  • Create a folder where you will run the model and a folder where you will put the grd, ini and bry files.
mkdir -p ~/ModNum/case_regional/CROCO_FILES
  • Download the croco pytools:
cd ~/ModNum/case_regional/
wget https://gitlab.inria.fr/croco-ocean/croco_pytools
  • Create a python environment (for example with mamba):
cd ./croco_pytools
module purge; module load mamba
mamba create -p ~/.mamba/croco_pyenv -f env.yml
mamba activate ~.mamba/croco_pyenv
  • Compile Fortran dependencies:
cd prepro/Modules/tools_fort_routines
make clean
make

Download the datasets

see tuto.download.html

We will need topography data, an ocean reanalysis (GLORYS form Mercator), and atmospheric data (ERA5).

The data for the Benguela region have already been downloaded and are available here /forums/public/pub/Data/.

For your project, you will have to re-download GLORYS and ERA5 data for your specific region using the scripts download_mercator.py and download_era5.py. Instructions are available in tuto.download.html. In particular, it will require you to create CMEMS and Copernicus accounts online.

Create the files

cd ~/ModNum/case_regional/croco_pytools/prepro
  • Download the file benguela.ini containing the requested user-defined parameters:
wget jgula.fr/ModNum/benguela.ini
  • Create the grid:
python make_grid.py benguela.ini
  • Create the initial file:
python make_ini.py benguela.ini
  • Create the boundary file:
python make_bry.py benguela.ini
  • You should now see the corresponding files:
ls ~/ModNum/case_regional/CROCO_FILES

croco_grd.nc
croco_ini_mercator_Y2024M01.nc
croco_bry_mercator_Y2024M01.nc

Compile and run the BENGUELA_LR case

  • Download and untar the croco code if you donâ€™t have it (see Activity 1).

  • We need the following files: jobcomp, cppdefs.h, param.h, croco.in so copy them into the folder you just created:

    cp ~/ModNum/croco/OCEAN/jobcomp ~/ModNum/case_regional/
cp ~/ModNum/croco/OCEAN/cppdefs.h ~/ModNum/case_regional/
cp ~/ModNum/croco/OCEAN/param.h ~/ModNum/case_regional/
cp ~/ModNum/croco/croco.in ~/ModNum/case_regional/

  • You might need to edit the jobcomp to specify the location of the source code and edit the cppdefs.h and param.h to run the model using openMP (see Activity 1).

  • We also need to modify cppdefs.h to use the ERA5 dataset:

#  define  ONLINE
#  define  ERA_ECMWF
  • Compile the code:
module load intel/12.1 netcdf/c-4.4.1.1-intel12 netcdf/fortran-4.4.4-intel12
./jobcomp
